## Teaching Assistant MVP

### Запуск проекта
Warning!!! Проект на данном этапе достаточно тяжелый, поэтому его стоит запускать на машине с хотя бы 16 GB оперативной памяти и хотя бы 50 GB места на диске, для сборки образов. Я в процессе уменьшения веса образов, но поверьте, я уже сделал много, чтобы сделать вес таким, какой он есть сейчас. Прошу понять и простить.

Для запуска проекта выполните следующие команды:
- `uv sync`
- `docker compose up`

В проекте работают следующие сервисы: 
- rag
- gen
- task_queue
- ingest_worker
- gen_worker

Что делает каждый из сервисов:

gen: расположен на `localhost:8000`. Имеет одну ручку `POST /questions/generate`, которая принимает на вход запрос для генерации вопросов, а также другие сопутствующие параметры, и возвращает сгенерированные вопросы.

rag: расположен на `localhost:8001`. Имеет одну ручку `POST /snippets/retrieve`, которая по запросу может достать релевантные сниппеты для query.

task_queue: расположен на `localhost:8002`. Является брокером сообщений. Имеет следующите ручки:
- `POST /gen/ping`: Отправляет воркера в сон на N секунд. Можно использовать для теста работоспособности очереди сообщений. 
- `POST /gen/questions/generate`: Создаёт задачу для генерации текста.
- `GET /gen/{task_id}`: Получает информацию о задаче с id {task_id}.
- `POST /ingest/ping`: Отправляет воркера в сон на N секунд. Можно использовать для теста работоспособности очереди сообщений.
- `POST /ingest/text`: Создаёт задачу по добавлению документа в RAG в формате JSON.
- `POST /ingest/plain`: То же самое, что и `POST /ingest/text`, но в текстовом формате.
- `GET /ingest/{document_id}`: Получает информацию о задаче по загрузке документа с id {task_id}.

В `localhost:6333/dashboard` можно посмотреть текущие коллекции Qdrant и их наполнение.
